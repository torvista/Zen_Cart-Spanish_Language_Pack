Installation

1) Prepare your development site.

As always, test install any modifications on a development site, NEVER on your real site.

a) The fileset is encoded in utf-8 as Zen Cart uses utf-8 encoding from v1.5 onwards.

So if you are using a new installation of Zen Cart 1.51 you should have no problem.

If you have an older site that still uses latin-1/8859 for the database contents/connection and html page encoding either you have to convert the database to utf-8 or convert the language files to 8859. This will be difficult as there are various free softwares available for bulk 8859->utf-8 recoding but not utf-8->8859.

I recommend you upgrade the database to utf-8. There are multiple threads on this subject in the Zen Cart forum but as of 2012-12 no script has yet been confirmed by the Zen Cart team as being 100% risk-free.

b) Your template

If you do not have a customised template or have been modifying the "classic" template: STOP NOW, create your own template and restore the "classic" files to their original states.
The "classic" template should be left untouched to help investigate/isolate any problems you create/introduce into the code with your customised template files.

c) Backup your files and database

If you have not installed the Backup MYSQL Plugin to provide easy database backups, do it now.
http://www.zen-cart.com/downloads.php?do=file&id=7

2) Install the files

a) Rename the "files_usted/YOUR_ADMIN" folder to match the name of your admin folder.

Use only the "files_usted" directory ("files_tu" has not been translated).

b) Copy all the folders under "files_usted" and copy them into your shop folder. This should not overwrite any existing files...

3) Add the Language into Zen Cart

Go into the Zen Cart Admin->Localization->Languages

Click on "New Language" and fill in the fields:

Name: This is a text field so can be anything eg: Espa√±ol/Spanish/Castellano. It can be changed anytime.
Code: es
Directory: spanish
Sort Order: Optional. The display order of the language selection drop-down in the admin menu and the display order of the flags in the shop front.
Set as default: tick if you want the admin to display in spanish when you login.

Click on "Insert".

If you get any warnings at this point MAKE A NOTE OF THEM/ take a screenshot and fix the problem: Google is your friend.

4) Check the installation

a) Check the Admin menus

Note that the text of the options of configuration can be stored in the database in one language only, and so are only available in english.

b) Check the Shop Front Sections

EZ Pages Menu

Problems

Untranslated Text/Mistakes/Improvements

If any text is untranslated/incorrect or could be improved post a message in the forum thread.

If you see a constant definition appear like TEXT_CUSTOM_ENTRY which shows the english text when english language is selected but the constant name when spanish is selected, first check that this define is not something that is part of a plugin you have added and so does not normally exist in a new Zen Cart installation (english) nor the spanish fileset.
If this is the case you have to find the define TEXT_CUSTOM_ENTRY in the english fileset and create an identical define in a corresponding spanish file.

Blank Screen/White Screen of Death

If you get white pages: check the debug log in the /logs (=>Zen Cart 1.51) or /cache (Zen Cart<1.51) directory and use this page to help you locate the fault: http://www.zen-cart.com/content.php?124-blank-page.

Translating the Zen Cart Installation Files

I decided not to do this as due to the almost total absence of spanish community-support for Zen Cart spanish-speaking users, anyone using Zen Cart will have to get used to reading english....starting with the installation.

If you find the install too dificult without spanish, then working with and customising Zen Cart will not be for you, give up now or start a spanish forum.
